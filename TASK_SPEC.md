# Технічне завдання: Моніторинг мережевих пристроїв

**Захід:** Студентський хакатон  
**Категорія:** Мережеві технології / Веб-розробка  
**Складність:** Середня

---

## 1. Загальний опис

Розробити програму або веб-сервіс для автоматичного моніторингу стану мережевих пристроїв з відображенням їх статусу в реальному часі та сповіщенням про відмови.

---

## 2. Функціональні вимоги

### 2.1 Обов'язкові функції (мінімальний рівень)

| # | Функція | Опис |
|---|---------|------|
| F1 | Опитування пристроїв | Автоматична перевірка доступності через ICMP ping або SNMP |
| F2 | Панель моніторингу | Веб-інтерфейс зі списком пристроїв та їх поточним статусом |
| F3 | Сповіщення | Повідомлення (UI, email або Telegram) при зміні статусу пристрою |
| F4 | Управління пристроями | Додавання, редагування та видалення пристроїв зі списку |
| F5 | Опис алгоритму | Текстовий або графічний опис обраного алгоритму опитування з обґрунтуванням ефективності |

### 2.2 Додаткові функції (підвищений бал)

| # | Функція | Опис |
|---|---------|------|
| D1 | Кольорова індикація | Візуальне розрізнення статусів: зелений — активний, червоний — недоступний, жовтий — нестабільний |
| D2 | Графіки доступності | Історія доступності пристрою у вигляді часового графіку |
| D3 | Карта мережі | Топологічна схема мережі з відображенням зв'язків між пристроями |
| D4 | Фільтрація та пошук | Пошук пристроїв за IP, назвою або типом |
| D5 | Статистика | Відсоток uptime за період (год/добу/тиждень) |

---

## 3. Типи пристроїв, що перевіряються

Система повинна підтримувати моніторинг щонайменше наступних типів пристроїв:

- Сервери
- Маршрутизатори (routers)
- Комутатори (switches)
- Принтери
- Комп'ютери / робочі станції
- IP-камери

Кожен пристрій у системі має містити:
- IP-адресу або hostname
- Назву (label)
- Тип пристрою
- Метод перевірки (ping / SNMP)
- Поточний статус
- Час останньої успішної відповіді

## 4. Алгоритм опитування

### 4.1 Вимога до документації алгоритму

Учасники **зобов'язані** описати та обґрунтувати обраний алгоритм опитування. Опис повинен включати:

1. Назву та принцип роботи алгоритму
2. Обґрунтування вибору (порівняння з альтернативами)
3. Оцінку часової складності та навантаження на мережу
4. Схему або псевдокод

### 4.2 Рекомендовані підходи (для натхнення)

- **Round-Robin polling** — послідовне рівномірне опитування всіх пристроїв
- **Adaptive polling** — зміна частоти опитування залежно від стабільності пристрою
- **Priority-based polling** — пріоритетне опитування критичних вузлів
- **Event-driven + periodic hybrid** — поєднання SNMP trap із плановим опитуванням

> Пошук та реалізація нових, більш ефективних алгоритмів заохочується та оцінюється окремо.

## 5. Технічні вимоги

### 5.1 Технологічний стек

Учасники обирають стек самостійно. Рекомендовані варіанти:

| Шар | Приклади |
|-----|---------|
| Backend | Python (FastAPI, Flask), Node.js (Express), Go |
| Frontend | HTML/CSS/JS, React, Vue |
| База даних | SQLite, PostgreSQL, Redis |
| Опитування | `ping` (ICMP), `pysnmp`, `net-snmp` |

### 5.2 Вимоги до інтерфейсу

- **Mobile-first** дизайн з адаптивною версткою (responsive)
- Коректне відображення на екранах від 320px до 1920px
- Елементи `section` та `div` — **без видимого контуру** (border/outline)
- Семантична HTML-розмітка
- Мінімальна кількість сторінок: 1 (дашборд), можна більше

### 5.3 Нефункціональні вимоги

- Час відгуку інтерфейсу — до 500 мс
- Автоматичне оновлення статусу без перезавантаження сторінки
- Зберігання мінімум 24 годин історії статусів

## 6. Критерії оцінювання

| Критерій | Максимальний бал |
|----------|-----------------|
| Реалізація обов'язкових функцій (F1–F5) | 40 |
| Якість та обґрунтування алгоритму опитування | 20 |
| Дизайн та зручність інтерфейсу (mobile-first) | 15 |
| Додаткові функції (D1–D5) | 15 |
| Код: читабельність, структура, коментарі | 10 |
| **Разом** | **100** |

### Бонусні бали

- +10 — оригінальний або покращений алгоритм опитування з доведеною вищою ефективністю
- +5 — реалізація карти мережі (D3) з автоматичним виявленням топології
- +5 — інтеграція зі сторонніми сервісами сповіщення (Telegram Bot, Email, Slack)
